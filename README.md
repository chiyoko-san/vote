# 世界一どうでもいい投票所（属性別分析つき）

全世界の人が、究極にどうでもいい二択にリアルタイムで投票するゲーム。
ただの暇つぶしに見えて、年代・性別・地方ごとの傾向が見えるので、地味に「データ」として面白い。

## 仕様
- お題は二択。タップで投票、投票後に結果バーが開示される
- **属性別フィルタ**：結果を「🌍全体／年代別／性別／地方別」で切り替えて見られる
- お題は**手動で「次のお題へ」**で全世界共有切替
- **お題は自分で投稿できる**（全世界のローテーションに追加）
- 1お題1票（変更可）

## 属性（プロフィール）について
- 初回に年代（10代刻み）×性別×都道府県を任意で入力。**スキップ可**
- プロフィールは端末内（localStorage）にのみ保存
- 投票時に送るのは集計用セグメント（年代/性別/地方）のみ。**氏名や個人特定情報は一切集めない**
- 都道府県は集計時に8地方（北海道/東北/関東/中部/関西/中国/四国/九州・沖縄）に丸めて比較

## 活用イメージ
- 「この二択、20代は割れるけど50代は一方的」みたいな世代差が可視化される
- 関東 vs 関西の食文化バトル（味噌汁は飲み物か、など）
- 自分で作ったお題でアンケートを取り、属性別の反応を見る
  → ちょっとした市場調査・ネタ収集ツールとしても使える

## 仕組み
- フロント：静的HTML 1枚（`index.html`）→ GitHub Pages
- バックエンド：Firebase Realtime Database（無料枠・サーバー管理不要）
- 票は `runTransaction` で競合なし

### DB構造
```
meta/currentId              現在表示中のお題id（全世界共有）
topics/{id}
  ├ a, b                    選択肢テキスト
  ├ votesA, votesB          全体の得票
  ├ order                   表示順
  ├ voters/{uid}            個人の投票（'a'|'b'）※重複防止用
  └ seg                     属性別集計（個人は特定不可）
      ├ age/{10..70}/{a,b}
      ├ gender/{male,female,other}/{a,b}
      └ region/{kanto,kansai,...}/{a,b}
```

## セットアップ

### 1. firebaseConfig を貼る
Firebaseコンソール → プロジェクト設定 →「マイアプリ」→ ウェブアプリ `</>` の `firebaseConfig` を
`index.html` の該当箇所に貼り替え。`databaseURL` は必須。
（THE BUTTONと同じプロジェクトを使い回し可。ツリーが別なので衝突しない）

### 2. データベースのルール
Realtime Database →「ルール」タブに `database.rules.json` の中身を反映。

### 3. GitHub Pages
新規リポジトリ（例 dumb-vote）を作って push → Settings → Pages で `main` / `(root)`。
公開URL：`https://＜ユーザー名＞.github.io/dumb-vote/`

## カスタム
- `PRESETS` … 初期お題
- 年代区分・地方区分は `AGE_LABEL` / `REGION_LABEL` / `PREFS` で調整可

## 注意
- お題投稿は自由（モデレーション無し）。荒れが心配なら投稿フォームを隠すか通報機能を追加
- セグメント別の票数が少ないお題は割合がブレる（サンプル数が出てくるまでは参考値）
